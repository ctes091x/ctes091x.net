## tl;dr
- Androidでは電通大のVPNに接続できない
- SSH Dynamic Port-Forwardingを使用すると学内ネットワークにアクセスできる
  - Termuxで`ssh -D 1080 x2500000@sol.cc.uec.ac.jp`を実行
  - Firefox BetaでSOCKSプロキシを設定


## はじめに

あなたは電通大生を膨大な課題から解放し真の自由をもたらすべく暗躍する地下組織のリーダーである。今日も調布市文化会館たづくりに借りた会議室に同志たちと集い、D棟奇襲作戦の立案を進めている。

電通大のD棟は1年次必修の実験科目「基礎科学実験A/B」で使用される。当該科目のレポートは電通大の1年生に多大なる負担を強いており、D棟の奇襲は電通大生解放の第一歩となるだろう。

作戦成功の鍵はD棟の構造把握にある。電通大施設課のホームページでは学内各建物の平面図がPDF形式で公開されている。あなたはD棟の平面図をダウンロードすべくスマホを開くが、そこではたと気づく。

「くっ！施設課のページは学内限定公開ではないか！」

学外から学内のネットワークにアクセスする際にはVPNを使用することが多い。電通大のVPNはL2TP/IPSecというプロトコルを使用しているが、Androidはバージョン12以降このプロトコルのサポートを打ち切っているため、あなたのスマホからでは学内ネットワーク限定のページを閲覧することができない。あなたのPCにはVPNがセットアップされているが、生憎今日は家に置いてきてしまった。

会合がお開きになることを覚悟したそのとき、ある同志の落ち着き払った声が届く。

「心配はいりません、同志」

静かに口を開いたのは、大学公認の技術系サークルMMAにも所属する、技術力と人格を兼ね備えた信頼厚き同志。

「Androidでも学内ネットワークにアクセスする方法はあります。**SSH Dynamic Port-Forwarding**を使うのです」


## Termuxで学内サーバにSSH接続

悠然とした口調で同志は続ける。

「同志、スマホにターミナルはインストールされていますか？」

「まさか！ 突然何を言い出すんだね」

とんでもないことを聞くものだ。いくら電通大の学生といえど、あらゆる端末でLinuxコマンドを実行して気持ち良くなっている者など少数派だろう。

「便利ですよ？ 1年次必修科目のコンピュータリテラシーの課題を片付ける助けになったのも一度や二度ではありません」

どうやら**Termux**というアプリを使用すると、特殊な権限設定なしにAndroidスマホ上で各種のLinuxコマンドを実行することができるようだ。Termuxの開発チームはGoogle Playストアからのインストールを推奨していないのだそうだが、今回はよしとしよう。

![Termuxのアイコン](https://avatars.githubusercontent.com/u/8104776?s=200&v=4)

![Termuxの画面]()

「では、TermuxでsolにSSH接続してください。このとき、コマンドに`-D 1080`というオプションを付加してください」

まだ`~/.ssh/config`に設定を記入していないので、solへのアドレス全体をタイプする必要がある。`ssh -D 1080 x2500000@sol.cc.uec.ac.jp`と打ち込みパスワードを入力すると、大学の教育用Linuxサーバsolに接続され、Bashのプロンプトが表示された。その様子に普段と違うところは全くない。

<!-- ![Termuxでsolに接続した様子]() -->

「それからどうするんだ？ まさか`curl`でページソースを取得して肉眼でパースするとでも？」

`curl`コマンドを使用すると、サーバーにHTTPリクエストを送信することができる。`curl https://shisetsu.office.uec.ac.jp/`といったコマンドで施設課のHPにリクエストを投げれば、レスポンスとしてそのソースが得られる。だが得られるのはあくまでソースだ。人間が読むようには書かれていない。

疑念に満ちたあなたの問いかけに、同志は応じる。

## Firefox BetaでSOCKSポートを指定

同志はさらに続ける。

「solにSSH接続するときに`-D 1080`というオプションを追加しましたね？ これはDynamic Port Forwardingと呼ばれるもので、SSHコマンドをSOCKSサーバとして振る舞わせ、TCP/IPの通信を中継させることができます」

「なるほど…… つまりページへのアクセスを学内のsol経由で行うと？ そのときにSSHコマンドがこの端末とsolとの間で通信の仲介をするわけか」

「ええ。既に学内のネットワークに接続されたマシンを踏み台にすることで学内と通信するわけです。VPNが自分の端末を仮想的に学内ネットワークに接続するのとは対照的ですね」

方法の概要は理解できたが、肝心な部分の説明はまだだ。ブラウザはTermuxのSSHコマンドのことなど気にもかけないだろう。このままでは通信はsolを経由しないし、学内のどこへもアクセスできない。あなたはさらに問う。

「これで完了というわけではないだろう？」

「その通りです。ブラウザからページにアクセスする際にSSHコマンドが仲介となるよう設定しなければいけません。必要な設定は単純ですが、生憎Androidデフォルトのブラウザはこれに対応していないので**Firefox**のベータ版を使います」

ベータ版のFirefoxもGoogle Playストアからダウンロードできる。安定版のFirefoxとは異なり、`about:config`でより詳細な設定項目を確認・変更できるようだ。以下の項目を変更した。

| 項目 | 値 | 説明 |
| --- | --- | --- |
| `network.proxy.type` | `1` | プロキシの種別を指定。今回は手動でのプロキシ設定を行うため`1` |
| `network.proxy.socks` | `localhost` | SOCKSプロキシのアドレスを指定。ここでは端末自身を指す`localhost` |
| `network.proxy.socks_port` | `1080` | SOCKSのポートを指定。SSH接続時に指定した`1080`を設定 |
| `network.proxy.socks_remote_dns` | `true` | ドメインの名前解決を接続先の側、ここでは学内で行うよう指定 |

「以上の通り設定すればFirefoxからの通信はsolを経由するようになり、従って学内限定のページを閲覧できるようになるはずです」

堂々と宣言する同志。ようやく問題は解決である。あなたの端末には、施設課が学内限定で公開しているD棟の平面図PDFが開かれていた。あなたはほっと息をつき、そして口を開く。

「さて、これでD棟の構造は把握できただろう。我々を苦しめた三大難実験であるところの比熱の測定・音の共鳴・エアトラックはいずれもD棟2階で行われており、同フロアには教員控室もも位置している。このすべてを同時に制圧することがD棟襲撃の要と———」
